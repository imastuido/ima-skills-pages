绘本漫画使用教程（fairytale-picturebook-pro）

本教程根据该技能目录下的 `SKILL.md`、`onboarding/*`、`references/*` 整理。目标是把童话绘本/漫画分镜生产流程标准化，确保角色风格一致、年龄适配、安全合规并可发布。

1. 启动前输入（最小必填）

  1. 场景类型：bedtime fairytale / emotional learning / nature exploration / festival story。
  2. 年龄段：`3-5`、`6-8`、`9-12`。
  3. 主题与情绪目标。
  4. 语言。
  5. 页数与发布渠道（book/social）。

如果范围过大，先降级为 4 页试点（pilot），确认故事方向后再扩页。

2. 五阶段工作流

  1. Onboarding：收集输入并确认范围。
  2. StoryTaskSpec：构建 `book_meta`、`story_outline`、`style_bible`、`character_bible`、`page_plan[]`。
  3. Generation：新页走 `text_to_image`，连续性修正走 `image_to_image`，偏差走 repair prompt。
  4. Result Handling：保存 RawResult，并标准化为 ArtifactResult，外链转稳定资产并记录 hash。
  5. Gates & Handoff：通过 QA/合规/预算与人工审批后，才进入发布交付。

3. 提示词模板（必须版本化）

每次生成都要记录：`prompt_version`、`anchor_version`、`scene_id`，保证可复现。

Master Prompt: [Style anchor] + [Scene setting] + [Character anchor] + [Action] + [Emotion] + [Lighting/material] + [Storybook constraints] Continuity Prompt: Keep the same [style/scene/character anchors]. Modify only [target delta]. Preserve [do-not-change list]. Repair Prompt: Preserve composition and narrative intent. Repair [character mismatch/style drift/prop errors]. Maintain [anchor list].

4. 年龄段写作约束

  • `3-5`：每页 1-2 句、冲突单一、词汇具体。
  • `6-8`：每页 2-4 句，允许轻量转折与因果。
  • `9-12`：允许复句与多角色互动，但避免成人主题。

硬性规则：不得出现血腥/羞辱/危险模仿引导；`age_readability_score` 不达标需改写。

5. QA 与合规发布门禁

  • 一致性门禁：角色、风格、场景锚点连续性。
  • 语言门禁：年龄可读性 + 风险语言扫描。
  • 可靠性门禁：重试状态、幂等冲突检查。
  • 成本门禁：`credit_used <= credit_budget_per_book`。
  • 人工门禁：高风险输出必须有编辑审批日志。

发布阻断条件包括：`policy_pass=false`、缺少平台 AI 披露字段、IP 风险未解除、应审未审。

6. 交付物清单

  • `story_outline.json`
  • `page_plan.json`
  • `style_bible.md`
  • `character_bible.md`
  • `artifact_result.json`
  • `publish_checklist_kdp.md`（目标渠道为 KDP 时）
